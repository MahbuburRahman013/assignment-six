Scope of var, let and const:

var :

In JavaScript, var is the function scope. That is, it has access inside the function in which it is declared.It has no block scope so it cannot be blocked by code blocker.It can redeclare variables and reinitialize them

let :

The "let" keyword has block scope, which means it is accessible only within the block of code in which it is declared.It does not have function-wide scope like "var."It has block scope so it can be blocked by code blocker.It can redeclare and can't reinitialize.

const :

The "const" keyword has block scope, which means it is accessible only within the block of code in which it is declared.It can't redeclare and also can't reinitialize.

use cases of null and undefined:

Both null and undefined in JavaScript mean that a variable's value is missing.But they are not the same.Undefined: A variable is "undefined" when it has been declared but has not been assigned a value.This usually happens when you declare a variable but don't set a value.
Or want to access an element of an array or object which does not exist.Null is a deliberate assignment to a variable to indicate that the variable has no value.Usually used to denote the value of a variable
